# Q-Sentinel IoT Access Control V4 Pro

Q-Sentinel is an ESP8266 NodeMCU based RFID IoT access control system. The project combines an RC522 RFID reader, a servo-based lock mechanism, IR optical key validation, a live web dashboard, a JSON API, and a Python/Qiskit BB84 quantum key distribution simulation.

## Key Features

- RFID UID reading with RC522
- Authorized and unauthorized card detection
- Servo-based lock control
- ESP8266 Wi-Fi web server
- Live modern web dashboard
- `/status` JSON API
- IR receiver based optical key validation
- Normal Mode: RFID card is enough for access
- Secure Mode: RFID card + valid IR optical key are required
- Manual Unlock and Reset Logs actions protected by a demo PIN
- Python/Qiskit BB84 simulation for the quantum-inspired secure communication layer

## Arduino IDE Usage

1. Open Arduino IDE.
2. Open this file:

```text
firmware/arduino/Q_Sentinel_V4_Pro/Q_Sentinel_V4_Pro.ino
```

3. Edit Wi-Fi credentials in:

```text
firmware/arduino/Q_Sentinel_V4_Pro/config.h
```

4. Select the board:

```text
NodeMCU 1.0 (ESP-12E Module)
```

5. Install the required libraries:

```text
MFRC522
IRremoteESP8266
```

6. Upload the firmware.
7. Open Serial Monitor at `115200` baud.
8. Copy the printed IP address and open it in a browser.

## VS Code / PlatformIO Usage

1. Open VS Code.
2. Install the PlatformIO extension.
3. Open this folder:

```text
firmware/platformio
```

4. Edit Wi-Fi credentials in:

```text
firmware/platformio/include/config.h
```

5. Build and upload from PlatformIO.

## Operating Modes

### Normal Mode

The system grants access when the authorized RFID card is scanned.

### Secure Mode

The system grants access only when both conditions are met:

```text
Authorized RFID card + Valid IR optical key
```

The IR key is valid for a configurable time window. The default value is 30 seconds.


Reset logs:


## Quantum-Inspired QKD Simulation

The repository includes a Python/Qiskit BB84 demo under:

```text
python/qkd_simulation/qiskit_bb84_demo.py
```

The simulation demonstrates the BB84 key distribution logic used in the project story. It is not real quantum communication hardware. It supports the concept of a secure communication layer for IoT status messages generated by the ESP8266 JSON API.

Run it with:

```bash
cd python/qkd_simulation
pip install -r requirements.txt
python qiskit_bb84_demo.py
```

## Security Note

This project is designed for education, prototyping, and demonstration. The dashboard PIN is not production-grade security. A real access control product would require HTTPS, proper authentication, secure key storage, backend authorization, tamper protection, and hardened power/control circuitry.
